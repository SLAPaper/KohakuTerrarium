# 多聊天分组（multi-chat groups）

KohakuTerrarium 的聊天面板可以像 VSCode 的编辑器组一样，被拆分为
多个并排的 *标签组*。每个组都是完整的聊天界面 —— 自己的标签条、
活动标签、消息流和输入框。把标签拖到另一个组的标签条上可以移动；
拖到组的边缘可以朝那个方向再切分。

当一个 terrarium 同时托管多个 creature 时，这个功能特别有用：

- 把 `developer` 钉在主组里，`reviewer` 和 `tester` 在旁边的组里
  实时滚动。
- 把 `ch:hub` 频道的转录停在生成它的 creature 旁边，方便核对
  广播内容。
- 在 `coordinator` 下指令，同时把 `searcher` 和 `analyst` 工作流
  分别放到独立的面板里。

多组功能 **始终可用** —— 没有开关、没有加入步骤。从不拖拽标签的
用户看到的就是一组一条标签列（与之前体验一致），但下面的手势
全程都是激活的。拖拽标签即可分屏。

## 拆分

每个聊天标签都支持两种拆分操作：

- **右键点击标签** → *Split right* / *Split down* / *Move to new
  group*。标签会被移动到新建的同级组里，原组保留其他标签。
- **拖拽标签** 到：
  - 另一个组的标签条 → 移动到该组；
  - 另一个组泡泡（bubble）的 *中央* 区域 → 同样（追加到末尾）；
  - 任意组泡泡的 *上/下/左/右* 25 % 边缘区域 → 该组朝那个方向
    切分，被拖动的标签放进新出来的同级组。

拖拽过程中边缘会高亮提示。中途按 *Escape* 可以取消。

## 关闭

- *Close tab*（标签上的 `×`，或右键 → *Close tab*）会从所有
  包含它的组里移除该标签。如果某个组因此变空，会自动收起，让
  同级组接管它的位置。
- 在组的空白区域右键 → *Close group* 移除整个组。如果只剩
  一个组，*Close group* 会回退为关闭多组模式。

## 键盘快捷键

| 按键 | 动作 |
| --- | --- |
| `Ctrl+\` | 把当前聚焦的组朝右切分。 |
| `Ctrl+Alt+\` | 把当前聚焦的组朝下切分。 |
| `Ctrl+W` | 关闭聚焦组里的活动标签。 |
| `Ctrl+Shift+W` | 关闭聚焦组（只剩一个组时会退出多组模式）。 |
| `Ctrl+Tab` / `Ctrl+Shift+Tab` | 按树顺序切换到下一/上一组。 |
| `Ctrl+1` … `Ctrl+9` | 聚焦到第 N 个组。 |

当焦点位于 input/textarea 内时快捷键会被让位给文本编辑（例如
`Ctrl+W` 仍然可以删除一个单词）。

## 持久化

聊天内部的分屏树是 *按 scope 区分、按机器存储* 的 —— 保存在
`localStorage` 的 `kt.chat.groupTree.<scope>` 键下，载荷包含
`version: 1` 字段，未来的迁移可以基于版本号判断。换一台机器
恢复同一会话时会从单组布局开始。会话历史本身保存在
`.kohakutr` 文件里，与布局无关。

## 暂不支持的功能

- **每个 scope 只有一条 WebSocket**：多个组共享同一个事件流；
  分屏只是显示层投影，不会在传输层再分叉。
- **branch 视图按标签共享**：两个组显示同一标签时，分支视图保持
  同步（与 VSCode "两个组里同一文件" 一致）。本期不支持 per-group
  branch override。
- **移动端 / compact 密度** 不进入多组模式，仅作为桌面特性。
- **悬浮窗**：暂不支持把组拖出成独立窗口。
- **跨组粘贴 / 转发**：目前用复制粘贴即可。
