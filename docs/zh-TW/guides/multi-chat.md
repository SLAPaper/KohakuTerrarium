# 多聊天分組（multi-chat groups）

KohakuTerrarium 的聊天面板可以像 VSCode 的編輯器群組一樣，被拆成
多個並排的 *標籤群組*。每個群組都是完整的聊天介面 —— 自己的標籤列、
活躍標籤、訊息流與輸入框。把標籤拖到另一個群組的標籤列上即可移動；
拖到群組的邊緣則可往那個方向切分。

當一個 terrarium 同時託管多隻 creature 時，這個功能特別有用：

- 把 `developer` 釘在主群組，`reviewer` 與 `tester` 在旁邊的群組
  即時滾動。
- 把 `ch:hub` 頻道的轉錄停在產生它的 creature 旁邊，方便核對
  廣播內容。
- 對 `coordinator` 下指令的同時，把 `searcher` 與 `analyst` 工作
  流程分別放在獨立面板。

多群組功能 **預設啟用** —— 沒有開關、沒有加入步驟。從不拖曳標籤的
使用者看到的就是一個群組、一條標籤列（與之前體驗一致），但下面的
手勢全程都是啟動的。拖曳標籤即可分屏。

## 拆分

每個聊天標籤都支援兩種拆分動作：

- **右鍵點擊標籤** → *Split right* / *Split down* / *Move to new
  group*。標籤會移動到新建的同層群組，原群組保留其他標籤。
- **拖曳標籤** 到：
  - 另一個群組的標籤列 → 移動到該群組；
  - 另一個群組泡泡（bubble）的 *中央* 區域 → 同樣（附加到尾端）；
  - 任意群組泡泡的 *上/下/左/右* 25 % 邊緣區域 → 該群組朝那個方向
    切分，被拖動的標籤放進新出來的同層群組。

拖曳過程中邊緣會高亮提示。中途按 *Escape* 可以取消。

## 關閉

- *Close tab*（標籤上的 `×`，或右鍵 → *Close tab*）會從所有
  包含它的群組移除該標籤。若某個群組因此變空，會自動折疊，由
  同層群組接管原本的位置。
- 在群組的空白區域右鍵 → *Close group* 移除整個群組。若只剩
  一個群組，*Close group* 會退回為關閉多組模式。

## 鍵盤快速鍵

| 按鍵 | 動作 |
| --- | --- |
| `Ctrl+\` | 把當前聚焦的群組朝右切分。 |
| `Ctrl+Alt+\` | 把當前聚焦的群組朝下切分。 |
| `Ctrl+W` | 關閉聚焦群組裡的活躍標籤。 |
| `Ctrl+Shift+W` | 關閉聚焦群組（剩下一個時退出多組模式）。 |
| `Ctrl+Tab` / `Ctrl+Shift+Tab` | 依樹狀順序切換下一 / 上一個群組。 |
| `Ctrl+1` … `Ctrl+9` | 聚焦到第 N 個群組。 |

焦點在 input/textarea 內時快速鍵會讓位給文字編輯（例如
`Ctrl+W` 仍可刪除一個單字）。

## 持久化

聊天內部的分割樹是 *依 scope 區分、依機器儲存* —— 存在
`localStorage` 的 `kt.chat.groupTree.<scope>` 鍵下，載荷帶有
`version: 1` 標記，未來的遷移可以依版本判斷。換另一台機器
還原同一會話時會從單群組佈局開始。會話歷史本身存在
`.kohakutr` 檔，不受佈局影響。

## 目前不支援

- **每個 scope 一條 WebSocket**：多個群組共用同一條事件流；分割
  只是顯示投影，不會在傳輸層再分流。
- **branch 視圖以標籤共享**：兩個群組顯示同一標籤時，分支視圖
  同步（與 VSCode「兩個群組看同檔案」一致）。本期不支援 per-group
  branch override。
- **行動裝置 / compact 密度** 不進入多組模式，僅作為桌面功能。
- **浮動視窗**：暫不支援把群組拖到獨立視窗。
- **跨群組貼上 / 轉發**：用複製貼上即可。
